ENVIRasterMetadata

This is a reference to a raster metadata object.

There are two ways to get a reference to an ENVIRasterMetadata object:

  • The ENVI::CreateRasterMetadata method returns a reference to a new, empty object that contains no metadata information.
  • An existing ENVIRaster object's METADATA property contains a reference to the populated ENVIRasterMetadata object associated with the ENVIRaster object.

Note: For an ENVIRasterMetadata object created with the ENVI::CreateRasterMetadata method, the band names metadata field may be added, removed, or updated. For an ENVIRasterMetadata object associated with an existing ENVIRaster object, the band names metadata field is always defined, and so it may only be updated.

The following table lists the special ENVIRasterMetadata fields that are used by the ENVI application:

ENVIRasterMetadata Editable Field Description 
       band names
Allows entry of specific names for each band of an image. 
       bbl
Lists the bad band multiplier values of each band in an image, typically 0 for bad bands and 1 for good bands. 
       class lookup
This field pertains to classification files and lists RGB color definitions for class names and each respective class. 
       class names
This field pertains to classification files and lists the classification names. 
       classes
This field pertains to classification files and defines the number of classes (including unclassified classes). 
       complex function
Specifies the values to calculate from a complex image and to use when displaying the image, calculating statistics for the image, or writing the image to a new file. Values include Real , Imaginary , Power , Magnitude , and Phase (the default value). 
       data gain values
Gain values for each band. 
       data ignore value

Pixel values that should be ignored in image processing. 

       data offset values
Offset values for each band. 
       default bands
If set, indicates which bands to display every time the file is opened. By default, a new image is automatically displayed when a file that has default bands defined in its header is opened. If only one band number is used, a gray scale image is loaded. 
       default stretch

Determines what type of stretch to use when the image is displayed. Values include No stretch , Linear 1% , Linear 2% , Linear 5% , Equalization , Gaussian , Square Root , Logarithmic , and Optimized Linear . 

       dem band
Path and filename of a DEM that you associate with an image. 
       dem file
Index (starting at 1) of a selected DEM band that you associate with an image. The dem band is not written if the DEM file contains a single band, or if the first band of an image was chosen. In these cases, the dem band value defaults to 0. 
       description
A character string describing the image or the processing performed. 
       fwhm
Lists full-width-half-maximum (FWHM) values of each band in an image. Units should be the same as those used for wavelength and set in the wavelength units parameter. 
       major frame offsets
The number of extra bytes to skip at the beginning and ending of the major frame. 
       minor frame offsets
The number of extra bytes to skip at the beginning and ending of the minor frame. 
       reflectance scale factor
The value that, when divided into your data, would scale it from 0-1 reflectance. 
       sensor type
Instrument types, such as Landsat TM, SPOT, RADARSAT, and so on. The available sensor types are located in the sensor.txt file (described in ENVI Classic Sensor File ). The sensor type code character tag defined here must match one of the entries in the sensor.txt file verbatim, including case. 
       spectra names
This field pertains to spectral library files only. It contains a comma-separated list of ASCII names enclosed in { } (curly brackets). 
       x start
      
       y start

Defines the image coordinates for the upper-left pixel in the image. Images that are spatial subsets of larger images often use an image coordinate system that references the parent (or larger) image so that you can link and dynamically overlay the two images. The default values for the upper-left pixel are (1,1).

Changing these values does not affect the way ENVI reads the image data from the file. 

       wavelength
Lists the center wavelength values of each band in an image. Units should be the same as those used for the fwhm field and set in the wavelength units parameter. 
       wavelength units
Text string indicating the wavelength units. Values include Micrometers , Nanometers , Wavenumber , GHz , MHz , and Index . If this field contains any other text upon querying, the value Unknown is returned. 
       z plot average
Values indicating the number of pixels in the x and y directions to average for ENVI Classic Z profiles. 
       z plot range
Values indicating the default minimum and maximum values for ENVI Classic Z profiles. 
       z plot titles
Allows entry of specific x and y axis titles for ENVI Classic Z profiles.

Example

The following code creates an ENVIRaster and prints all available metadata tag names and values.

; Launch the application

e = ENVI()

 

; Create an ENVIRaster

file = FILEPATH('image.tif', $

  SUBDIRECTORY=['examples','data'])

raster = e.OpenRaster(file)

 

; Print all metadata values

metadata = raster.METADATA

PRINT, metadata ; print tag names and values

PRINT, metadata.TAGS ; print string array of tag names

PRINT, metadata['lines'] EQ raster.NROWS ; true

 

; Add, then update, a user-defined item

metadata.AddItem, 'Author', 'EXELISVIS'

metadata.UpdateItem, 'Author', 'Exelis Visual Information Solutions'

 

; Update a format-defined item

metadata.UpdateItem, 'band names', $

  ['Blue', 'Green', 'Red', 'NIR']

 

; Print the updated metadata values

PRINT, metadata

 

; Remove an item

metadata.RemoveItem, 'Author'

PRINT, metadata

Syntax

Result = ENVIRaster.metadata

Use brackets to retrieve metadata values, using the following syntax. Type the metadata field exactly as it appears in the above table.

var = metadata [' field name ']

Methods

AddItem

RemoveItem

UpdateItem

Properties

Properties marked as (Get) can be retrieved, but not set.

COUNT (Get)

A long integer that represents the number of available tags.

TAGS (Get)

A string array of tag names used to retrieve metadata items. Metadata tags are defined by the file format from which the ENVIRaster was opened. Consult your data provider or file format specification for more information on metadata fields for specific sensors and/or data formats (GeoTIFF, NITF, and others).
