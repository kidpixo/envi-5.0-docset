ENVI::GetTemporaryFilename

The GetTemporaryFilename function method returns the full filepath of a file located in ENVI 's temporary directory. This filename is unique in the filesystem.

Example

; Launch the application

e = ENVI()

 

; Create an ENVIRaster

file = FILEPATH('qb_boulder_msi', ROOT_DIR=e.ROOT_DIR, $

SUBDIRECTORY = ['data'])

raster = e.OpenRaster(file)

 

; Create an output raster

tempFile = e.GetTemporaryFilename()

PRINT, tempFile

 

; Retrieve data from original raster that contains only a single band

origData = raster.GetData(BANDS=0)

 

; Create raster and save it in tempFile

newRaster = e.CreateRaster(tempFile, origData, NBANDS=1)

newRaster.Save

 

; ... Use newRaster and the output raster file

; as you see fit, and then ...

 

; Delete the output raster file when we're done with it

newRaster.Close

FILE_DELETE, tempFile

Syntax

Result = ENVI .GetTemporaryFilename([ Extension ] [, ERROR= variable ])

Return Value

This method returns a string containing the full filepath of a file in ENVI 's temporary directory.

Arguments

Extension

A scalar string denoting the filename extension. If this argument is not supplied, the .dat extension is appended.

Keywords

ERROR

Set this keyword to a named variable that will contain any error message issued during execution of this routine. If no error occurs, the ERROR variable will be set to a null string ( '' ). If an error occurs and the routine is a function, then the function result will be undefined.

When this keyword is not set and an error occurs, ENVI returns to the caller and execution halts. In this case, the error message is contained within !ERROR_STATE and can be caught using IDL's CATCH routine. See IDL Help for more information on !ERROR_STATE and CATCH.

See Manage Errors for more information on error handling in ENVI programming.
